# Copy Code Fragment LN 设计说明

## 目标

提供一个 VS Code 扩展，在用户选中代码后：

- 可通过编辑器右键菜单执行复制
- 可通过默认快捷键 `Ctrl+Shift+C` 执行复制
- 复制内容仅包含当前文件的相对路径和选区行号范围

## 输出规则

- 单选区：`相对路径:起始行-结束行`
- 多选区：`相对路径: 选区一 选区二 选区三`
- 多选区之间使用空格分隔
- 行号为 1 基

## 路径规则

- 路径相对工作区根目录计算
- 统一输出为 `/` 分隔，避免 Windows 反斜杠影响复制结果

## 交互规则

- 仅在编辑器有选区时显示右键菜单
- 仅在编辑器有选区时允许快捷键触发
- 无选区时不展示入口
- 如果用户从命令面板强制触发但上下文不满足，则给出友好提示

## 特殊情况

- 多选区仅考虑同一文件内的多个选区
- 对于 `file` 以外的资源类型暂不支持
- 对于不在工作区内的文件暂不支持

## 技术方案

- 使用一个命令 `copyCodeFragmentLn.copyLocation`
- `package.json` 同时贡献命令、编辑器右键菜单和快捷键
- 在命令实现中读取活动编辑器、工作区相对路径和全部选区
- 使用 VS Code 剪贴板 API 写入结果

## 测试策略

- 纯函数测试：行号范围计算、单选区和多选区格式化
- 扩展集成测试：打开工作区内文件、创建选区、执行命令并校验剪贴板内容
