# Copy Code Fragment LN 快捷键可配置设计说明

## 目标

让插件在保留默认快捷键的同时，明确支持用户按照 VS Code 的标准方式自定义快捷键绑定。

## 事实约束

- VS Code 扩展可以在 `package.json` 中贡献默认快捷键
- VS Code 扩展不能可靠地通过插件设置项动态修改快捷键绑定
- 用户自定义快捷键的标准入口是：
  - `Keyboard Shortcuts`
  - `keybindings.json`

## 方案选择

采用以下方案：

- 保留现有默认快捷键贡献
- 不新增“快捷键设置项”
- 优化命令标题和说明，让用户更容易在快捷键搜索界面中找到该命令
- 在 README 中明确给出自定义快捷键的方式与示例

## 用户体验

- 插件安装后开箱即用，默认快捷键仍然有效
- 用户若想改键位，可直接在 VS Code 中搜索命令后重新绑定
- 用户若偏好手动维护配置，可直接编辑 `keybindings.json`

## 代码范围

- `package.json`
  - 保留 `contributes.keybindings`
  - 视情况调整命令标题或分类文案
- `README.md`
  - 增加“自定义快捷键”章节
  - 给出 `keybindings.json` 示例

## 不做的事情

- 不增加一个不会真正生效的快捷键设置项
- 不尝试在运行时劫持或覆盖用户的 VS Code 快捷键配置

## 验收标准

- 默认快捷键继续生效
- 用户能在 VS Code 的快捷键面板中搜索到该命令
- README 中包含清晰的快捷键自定义说明
- 现有测试继续通过
